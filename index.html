Seed CSS

A light-weight, mobile first and responsive CSS boilerplate.

DOWNLOADv1.0.4

(This page design is proudly based on seed-css)


Why you're gonna love it?

It's made for modern browsers, not intrusive and lightweight.


Easy to start

Get started to your projects with a very small footprint CSS boilerplate like this.


Light-weight

Just 11kb of CSS your app needs to load for a mobile first and responsive design.


Based on Sass

All written on Sass, what means you can easily customize it as much as you need.

Get started

Just before get started, please notice that seed-css was written with modern browsers in mind. That means it just has support for IE9+ (which actually isn't a real modern browser ... but), Chrome, Firefox and Safari, besides mobile browsers. If you need support older browsers, this boilerplate may not help you, but, just in case, you're free to customize it and create your own CSS rules to make it supports those older browser versions.

The most first thing you need to do to start using seed-css is import its CSS file into your pages. In order to get the right files, you can either fork this project on Github, or just download the compressed project files to your computer.

After get the latest version of files, import it to your pages as the example below: 

<head >
  ...
  <link href="seed.css" rel="stylesheet" type="text/css" >
</head>

Once you have the seed-css stylesheet embeded to your site or application, have a look on all code examples below to get inspired about what features it offers you and what you can do with it. :)

Class Detail Remarks
.grid   Defines the grid container
.row   Defines each row of a grid container
.col   Defines each column of a grid container
.sm-* (1 ~ 12) Defines a column size for small screens. It's used concatenated to .col
.md-* (1 ~ 12) Defines a column size for medium screens. It's used concatenated to .col
.lg-* (1 ~ 12) Defines a column size for large screens. It's used concatenated to .col
.xl-* (1 ~ 12) Defines a column size for extra-large screens. It's used concatenated to .col
*-offset-* (sm, md and lg) and (1 ~ 12) Creates an offset of columns. It's used concatenated to .col and its screen size. E.g: .md-offset-2.

    <div class="col sm-12 md-6 lg-3" > ... </div>
    <div class="col sm-12 md-6 lg-9" > ... </div>
    </div>
</div>

Media queries

Media queries allows us to make an even more accurated design. In seed-css they are made to represent the following screen sizes:

  Screen width Prefix Columns Container
Mobiles < 600px .sm 12 90%
Tablets <= 990px .md 12 85%
Desktops/ laptops <= 1200px .lg 12 80%
Extra large desktops/TVs > 1200px .xl 12 75%

You can safely write your own styling rules using the following queries and keep covered by seed-css grid system. Using a different set of media queries may overwrite this boilerplate grid system.

Query Remarks
@media only screen and (min-width: 0px) For mobile devices. (Mobile first)
@media only screen and (min-width: 600px) For tablets devices.
@media only screen and (min-width: 990px) For desktop/ laptop devices.
@media only screen and (min-width: 1200px) For extra large desktop/ TV devices. 
// Mobile first
@media only screen and (min-width: 0px) {
  ...
}

}

// Desktops/ Laptops
@media only screen and (min-width: 990px) {
  ...
}

// Extra large desktops/ TVs
@media only screen and (min-width: 1200px) {
  ...

Icons

Class Detail Remarks
.btn   Makes anchors looks like buttons
.btn-primary   Defines a 'primary' (blue) colored button
.btn-(*color*)   Defines a colored button. Colors can be: red, orange, green or purple
.btn-(*size*)   Defines a sized button. Sizes can be: small or medium. Default is big
.disabled   Defines disabled buttons. Simply setting the disabled property will also make button looks like disabled
.flat-btn   Defines flat buttons, which means no borders or background colors. You are still able to use .btn-(*color*) and or .btn(*size*) on this button type

Exceptionally the file upload component works better with a javascript support. Using javascript will create a much better user experience. To make it easier seed-css has a plug-and-play javascript library that you can use, as follows:

When using Node JS:
// Import 
var fileUpload = require('seed-css/seed-css.js');

// Initialize 
fileUpload();
When using Vanilla JS:

First download the javascript library here.

Then, include the following code to your HTML page, right before close the BODY tag.

<script src="seed-css.js" ></script>

if (typeof fileUpload !== 'undefined') {
  fileUpload();
}

Modals

Modals are an essential visual component for a best user experience, allowing the website either to make interactions to its visitor or request imediate feedback.

Hello world

This' the body of my overlay modal window.

<!-- The trigger -->
<button type="button" role="modal" data-modal-id="modal" class="btn-primary" > 
  Open modal
</button>

<!-- The modal -->
<div style="display: none;" class="modal hidden" >
  <div class="modal-window" >
    <div class="modal-header" >
